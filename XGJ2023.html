XAMK Game Jam Spring.

  • April 02, 2023

Project description

The game was made as part of XAMK Game Jam 2023, which was organized from 31 March to 2 April 2023. The team included Tarita Karhulahti, Riku Heino, Iina Andersson and Konsta Karttunen. On the first day we started to plan our game and we decided to do FLAME ON, SPARKY! Players need to find and bring back all the missing flames that had disappeared from the bonfire. Player has x minutes time to collect all the missing flames before times runs out.

Description of the task

My task in the team was to be one of the programmers in the game jam. I was responsible for making the UI, animation and game level. I also added music and sound effects to the project.

Description of methods

In the game, the player moves the character using keyboard WASD. Main event is find all missing flames and bring them back the bonfire. There is also enemies who try to destroy player. 


    public class CollectedFlamesUI : MonoBehaviour
    {
        public int flameNumber;

        public int numOfFlames;

        [SerializeField]
        private Image[] flame;

        [SerializeField]
        private Sprite fullAmountOfFlames;

        [SerializeField]
        private Sprite emptyFlame;

        private void Update()
        {
            if (flameNumber > numOfFlames)
            {
                flameNumber = numOfFlames;
            }

            for (int i = 0; i < flame.Length; i++)
            {
                if (i < flameNumber)
                {
                    flame[i].sprite = fullAmountOfFlames;
                }
                else
                {
                    flame[i].sprite = emptyFlame;
                }
                if (i < numOfFlames)
                {
                    flame[i].enabled = true;
                }
                else 
                {
                    flame[i].enabled = false;
                }
            }   
        }

From the code above, you can see flame collector.

Problems and solutions

Our team faced significant challenges with communication and task delegation. As a result, each member ended up working on different aspects without a clear division of responsibilities. This caused unnecessary dependencies and forced some team members to fix issues that could have been avoided. Overall, improving communication and organizing tasks more efficiently could have helped us avoid these issues.

Final assessment

The game we developed turned out to be incredibly fun and enjoyable. I had the opportunity to explore new features like the custom shader terrain tool, and my team members were helping throughout the development process. Working with them allowed me to learn new skills and techniques.
